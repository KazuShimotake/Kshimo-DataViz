2.1 Figuring out what HTML elements had to do with linking

One piece to figuring out the linking was to understand how paths work, or so we thought. This led us to writing down our plan and goals relating to how we wanted to solve the linking problem.

We were also brainstorming how path elements could relate to linking the two graphs together since we were not sure what path elements had to do with linking in the first place. So as we were trying to link things up we realized that we needed to have one country get mapped to one or many cats and also have one cat map to one or many countries. This led us into the next part.

2.2 Mapping a one to many connection through class and id

So we went to office hours to ask more about how to link up a one to many relationship and our professor suggested we try to set the class of a circle element to all the countries the cat was from and to set the id of the paths to the country name.
This led us to think about how class and ids relate to linking the two graphs together. So we went back to the white board thinking about how to link the different elements together so that when you highlight (ie. mouse over) one element in one SVG, it changes the attributes of another element in the other SVG.

It was around here that we realized a variety of things:

  1. How the js Map object was going to allow us to get access to rows of the table we were interested in.
  2. How the different elements were encapsulated and modifying path elements did not affect circle elements. Also we need to keep track of which pieces belonged to which SVG.
  3. How selection allowed us to change HTML elements dynamically
  4. That the path element was no different from other elements and thus we can modify it in the same way we are modifying the circle elements for the scatter plot.
  5. How we can join data to an HTML and how to acess that data.
  6. How to get access to the data stored into HTML by using JS maps.
  7. That we need to create a separate CSV and Map for countries and the cat breeds that are associated with that country.

Funny thing about all these relizations was their connection to our understanding of the roles of classes and ids for HTML and Js.

2.3 Figuring out the role of classes and ids

Of the many things we had figured out, realizing how the classes and ids was essential. We say this because the moment we realized the role of classes and id, all the other code we had set up suddenly fell into place.

The roles for ids,

  • Was used to get the column from the correct cvs stored in a JS Map and set it up as the class of the element that triggered the event
  • Was to be both the key in a map that contains data, and to be an identifier for an HTML element.

The roles for classes,

  • To be a list of values we can use to find HTML elements we can modify dynamically

In figuring out the roles we also discovered how to leverage the properties of classes in a way that allow us to create a one to many link:

  1. One HTML element can have multiple classes
    • So one circle element can have multiple countries associated with it
    • One path element can have multiple cat breeds associated with it
  2. Ids have to be unique, whereas classes do not.
    • So multiple different elements can have the same class.
    • So multiple circle elements can be associated with the same country/countries
    • So multiple path elements can be associated with the same cat breed/breeds.
  3. When you search for a class it looks for HTML elements that have at least one instance of that class listed. So it will return an HTML that has just that class or an HTML that has that class plus more different classes.

    searchAll(.usa) returns the following:

    • <circle id =”cat breed 1” class = “usa”></circle>
    • <circle id =”cat breed 2” class = “usa canada”></circle>
    • <circle id =”cat breed 3” class = “china usa japan”></circle>


    searchAll(.usa.canada) returns the following:

    • <circle id =”cat breed 1” class = “usa canada”></circle>
    • <circle id =”cat breed 2” class = “usa russia canada”></circle>
    • <circle id =”cat breed 3” class = “canada england wales usa japan”></circle>

So with our new understanding of the roles of ids and classes and the properties of classes we were finally able to actually set up the HTML elements so that the linking would work.

2.4 How to set up each HTML element so that the linking would work

The map object required a key and that key was the cat breed. This then gave us a row from the cat data set. We could then select the country column and assign it as the class of that circle element with the join().

Seeing how this worked for the circles and cat breeds we then thought ok so we just need to do this for the path objects. However, our Map only took cat breeds as the key. We also needed to use one country to find one to many cats.

To do this, we had to go back to Google Colab and create a separate CSV that had countries, the number of cats breeds from those countries, and a list of cat breed names. Using this new CSV, we made another map object that had countries as keys and a list of breeds from each country as values.

(The map with 34 items is the country to cats map.The one with 61 is the map of cat breeds which includes the list of countries they’re from)

Then we repeated the same process as above, this time making the class of the paths to be the names of cat breeds and the id the name of the country.

  • The way we set up the classes and ids allowed us to map a one to many relationship
  • Classes need to be separated by spaces

From Map to Scatter Plot:
  • We highlight a country, USA
    • We get the id USA
    • Then in countryData we use the key USA, to get the value, a row, in countryCatdata
    • Then for that row we look for an attribute/ column called names ( ie. cat breeds)
    • We then get that cell value which will be a list of names.
    • We then compare that list of names with the classes of the circles in the scatter plot. ( that is we searchAlL of the <circle>s that have a class that matches the list of names for the USA.
      • If the class of a circle element matches the list of cats then do something with this circle, like make it bigger, or change its color.
      • If the class of this circle element doesn’t match then do not do anything.

  • We highlight a dot in the Scatter plot, Cornish Rex

    • Then for that row we look for an attribute/ column called country
    • We then get that cell value which will be a list of countries the cat breed originates from.
    • We then compare that list of countries with the classes of the path in the map. ( that is we searchAlL of the <path>s that have a class that matches the list of countries for the Cornish Rex.
      • If the class of a path element matches the list of cats then do something with this path, like change its color.

Iteration After Feedback:

So around a week after working on our project, the professor had us demo and evaluate the current state of our projects so that we could improve and iterate on them. This is an important step of the design process because it allows us to challenge our assumptions and consider areas where we are not as doing as well as we think we are. It allows us to incorporate others perspectives while at the same time learn about those perspectives.

Feedback Activity:

In class, Ben (a.k.a. the professor) had us test out each other’s projects and give constructive feedback about how well the visualizations enabled specified tasks such as “discover what cats are from a country”. He had us list out three good points that worked well, two bad points that made the task difficult, and at least one suggestion for improvements on each other's visualizations.

Feedback Summary:

So after collecting all our feedback, these are the common themes:

Comments Relating to the Scatter Plot:

  • Hard to find info about specific breeds
  • Hard to understand because of overlapping
  • Don’t see correlations
  • Would be nice to be able to adjust the scatter plot to different x and y axes

Suggestions for the tool tip:

  • Tooltip is difficult to read when the list of breeds from a country is too long
  • Tooltip would be useful for differentiating different cat dots on the scatterplot
  • Suggested information to add to scatterplot tooltip:
    • Name and country
    • Pictures of the different cat breeds

Why do we think this feedback is constructive or not:

Scatter Plot improvements:

  • The feedback about finding info about specific breeds in the scatter plot is helpful since it directly interferes with the tasks that prospective viewers might want from our visualization.

Tool tip suggestions:

  • The same goes for the difficulties around the tooltip for countries with long cat lists and not having the tooltip for the scatter plot dots having name and country
  • The suggestion for pictures in the scatter plot tooltip is something that could be interesting, but it would also take up more space when it pops up and potentially block out access and visibility to other dots

The feedback about overlap and correlation is more related to the data and data processing than the scatter plot visualization itself...

Overlap:

  • For this we can try to process the data with litter alteration (i.e. add jitter ) or change the opacity of the dots( which we already have done), change the size of the dots, make it so we can interact with the stack of dots. Beyond that there is not much we can do about overlapping. We do not want to modify the data values too much to preserve it.

Correlation:

  • For correlation there is not much we can do about correlation if the data doesn’t correlate. What we can do is make it so the user can change the x and y axis so they can look through different attributes of the data and find trends in the data.

Augmentations based on this feedback:

Scatter Plot:

  • We added minor jitter to our data to reduce overlap of cat breeds
  • We added dropdown menus for the axes to be able to display relations between different traits such as minimum or maximum weight
  • We added a tooltip popup for the scatter plot side that states a dot’s cat breed and the country or countries that cat is from
    • When the popup comes up for a cat dot, the cat’s name is bolded while the countries that it’s from are normal text
    • Similarly, when it comes up for a country, the country’s name is bolded while the cats from that country are normal text

Tool Tip:

  • We formatted the display of the tooltip for long lists so it would be easier to read.
  • For the map we bolded the country and for the scatter plot we bolded the cat breed name.
